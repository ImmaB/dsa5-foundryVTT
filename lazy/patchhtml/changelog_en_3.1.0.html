3.1.0 Important Enhancements/Notes (1/13/2022)

If your Foundry installation behaves quirky try this fix provided by Foundry:

Player menu

There is now a player menu available next to the game master menu. As the name says this is primarily intended for players to assist in certain complicated procedures. The first one available will be the summoning of creatures (see below). Future updates could include tools and rules to create items, alchemy and other construction kit-like tasks.

Summoning

The player menu now includes the possibility to summon creatures. The base system has rules for elementals and demons. Other creature categories will be added via different modules later on. This tool allows players to assemble summons quickly via a construction kit which respects the relevant rules and shows possible modifications. After finishing the construction plan, the players can send it to the game master who then can decide to push the creation as a token onto the map. To do so the player needs at least observer permission for the summoned creature and drag it onto the summoning application and then roll the summoning spell which is automatically modified by the conjuring modifier of the creature. The app will show you available quality levels that you can then use for additional services or spend to add modifiers and traits to the creature.

Light Macro Improvement

The light macro included in the core module has been reworked completely. Among the new features you can find the following:

  • Light controls can be enabled and attached to the token HUD. After enabling the setting you will find a submenu on the token to enable all the lights directly.
  • The presets for light and vision settings can now be altered
  • Light and vision presets can be modified with multiple automatic factors to support different playing styles. You can have a general factor for every light and vision setting applied through the macro. One attached to each scene differently. And finally one which recalculates the vision range depending on the darkness setting of the scene. This means your players can see less the darker it gets. Keep in mind that the factors are also added to the token HUD light settings.
  • Moreover there is a new "auto" vision mode which detects whether the token has darksight and similar vision -ltering modifiers and then automatically applies a best guess for the active vision settings.
  • Finally you can automate the vision settings altogether so they get updated every time on entering new scenes or changing the darkness setting. While this is a useful and strong new feature, it will also most likely require some more fine tuning in future patches.

Enhancements

  • Regeneration (or degeneration) of AE, KP, and LP at the end of the round is now possible (e.g. for vampires)
  • Actors can now have effects for permanent spell damage increase
  • Category "Magical sign" is now available
  • Category "Pact Gifts" is now available
  • The -2 modifier for unfamiliar spells can now be activated via the configuration. It'll propose a modifier and put a warning that spell modifications are disallowed. You can still override both proposals if you need to do so. The decision whether the spell is included in your tradition is taken from a comparison between the tradition field of the actor sheet and the distribution field of the spell. So make sure to include the necessary traditions there!
  • The calculation of damage caused by breaking point rating checks has been altered a little bit. The damage is now round up if the structure points are not dividable by four. So damage to a shield with 30 structure points prepatch was 30/4 rounded down = 7. After the patch it's 30/4 rounded up = 8.
  • Paying and demanding money via the game master menu now deducts or delivers the money immediately without the need of a button press in the chat. This allows to transfer money to actors who are not primarily connected to a player. The chat macro /pay and /getPaid still work the same way.
  • Actor links in journals should now open already imported actors if they reside in the automatically generated folders. If they are unavailable there, the compendium entry will be opened instead.
  • Ammunition transfers its domain attributes (e.g. magical, blessed) to the ranged weapon during a shot, making the attack magical or blessed.
  • The system can now use Rollsmiths' dice so nice custom dice systems. You can set one system for every dice type (e.g. for Courage, Attack). See configuration of TDE.

Bugfixes

  • Several compatibility adjustments to TDE modules
  • Proper table for liturgical chant botches should be shown
  • Profession/Species/Culture wizards add abilities which depend on other abilities a little bit better
  • Hotkey for next turn can not be used by players anymore unless it's their turn
  • Advantage "Tough Animal" calculates pain resistance properly
  • Compatibility to module "multilevel-tokens" improved
  • Compatibility to monarch card sheet
