2.3.0 Wichtige neue Features (5.10.2021)

Active Effects Helper Item

SLs können nun einen speziellen Gegenstand namens "Effekt-Container" erstellen. Der einzige Zweck dieses Gegenstandes ist die Speicherung von Aktiven Effekten, da Foundry keine Standardoption dafür bietet. SLs können mehrere Aktive Effekte zu diesem Element hinzufügen. Diese Effekte werden beim Drag and Drop in einen Actor kopiert, was für häufig verwendete Effekte nützlich ist. Dieser Workaround könnte in Zukunft entfernt werden, sobald Foundry eine bessere Basisimplementierung dieser Funktion hat.

Gruppenschicksalspunkte

Das Menü des Meisters enthält jetzt Gruppen Schicksals-Punkte. Diese können wie normale Schicksalspunkte über das Chat-Kontextmenü verwendet werden. Schicksalspunkte, die man für den Pool verwenden will, muss man manuell aus den Actors entfernen. Du kannst dies tun, indem du einen negativen Modifikator zum maximalen Schicksalspunktewert auf dem Charakterblatt hinzufügst (dies kann nicht automatisch verfolgt werden, da das Tool nicht weiß, welche Charaktere dem Pool Punkte hinzufügen).

Waffenvergiftung & Talisman

Du kannst nun deine Nahkampfwaffe vergiften. Ziehe dazu ein Gift auf das geöffnete Gegenstandsblatt einer Nahkampfwaffe. Bei einem erfolgreichen Angriff erscheint ein Button im Chat, um die Giftprobe auszuführen. Jedes Mal, wenn du die Giftprobe bestehst, wird eine Ladung des Giftes entfernt. Setzt die Option "Permanent", wenn ihr nicht wollt, dass bei der Verwendung von Gift Ladungen abgezogen werden (zum Beispiel bei feindlichen Kreaturen oder Spezialwaffen). Auch das Erstellen von Talismanen ist nun auf dieselbe Weise möglich wie das Verzaubern. Der einzige Unterschied besteht darin, dass sie sich nie verbrauchen, wenn sie auf permanent eingestellt sind, und dass sie nicht wieder aufgeladen werden müssen.

Verbesserungen

  • Der Parierwert von Waffen wird nun über das primäre Attribut der Waffe berechnet und nicht mehr über die Kampffertigkeit. Man kann nun spezielle Waffen mit z.B. Scharfsinn als primäres Attribut erstellen.
  • Das Anwenden von Schaden über das Chat-Kontextmenü hinterlässt nun einen kleinen Indikator, der den Spielleiter daran erinnert, dass der Schaden bereits angewendet wurde.
  • Actor Flags werden bei der Erstellung von Actors nicht mehr gelöscht, um Module von Drittanbietern besser zu unterstützen.
  • Fehlende Initialisierungstexte für Abenteuer sollten nun wieder korrekt angezeigt werden.
  • Würfel können nun mit individuellen Dice so nice Farbsätzen im Konfigurationsmenü konfiguriert werden.
  • Die optionale Regel "weniger Regeneration" kann nun über die Konfiguration aktiviert werden.
  • Der Journalbrowser verfügt nun über einen kleinen Pfeilbutton zum Ausblenden des Inhaltsverzeichnisses. Dies kann die Lesbarkeit von großen Textblöcken verbessern.
  • Man kann nun permanente und temporäre Modifikatoren zum Regenerationswurf hinzufügen. Um dies zu tun, suche deine LP-Konfiguration auf dem Attribute/Spezialfähigkeiten-Teil des Charakterbogens und klicke auf den Namen. Dadurch wird das entsprechende Menü geöffnet. ACHTUNG: Temporäre Modifikatoren werden beim nächsten Regenerationswurf gelöscht.
  • ActiveEffects können nun Gruppen von Fertigkeitsprüfungen modifizieren (z.B. liturgische Gesänge, Zauber).
  • Normale Würfe über Chat-Befehle haben nun eine Kontextoption, um Schaden zuzufügen (z.B. wenn ein Spieler zufällig vom Blitz getroffen wird und man über den Chat Schaden zufügen möchte).
  • Der Kampfreaktionsdialog wurde optimiert, um die Anzahl der erforderlichen Klicks um eins zu reduzieren. Deine Maushand wird es dir danken.
  • Zaubersprüche zeigen nun die Wirkdauer im Charakterblatt und im Kampf-Tracker an.
  • Die Tokenleiste zeigt nun immer mindestens einen Zauber im Kampf an (wenn der Charakter ein Gesegneter oder Magier ist). Diese Schaltfläche kann zu einem Untermenü erweitert werden, in dem alle Zaubersprüche angezeigt werden. So hat man Zugang zu Zaubern, die nicht nur für Schaden gedacht sind (z.B. Axxeleratus).
  • Die Zustände Blutung und Verbrennung wenden nun automatisch in jeder Runde Schaden an (nur im Kampf!).
  • Das Hauptmenü kann nun einen Standard-Sichtmodifikator auf Kampfwürfe anwenden, der von der Dunkelheit der Szene abhängt (Dunkelsicht wird automatisch in die Berechnung einbezogen).
  • Das Händlerblatt hat jetzt eine Option für ein einfaches Charakterblatt für epische und legendäre Helden (wie einen König oder deinen Ehepartner), die keine wirklichen Werte haben und nur ein Bild und einige Hintergrundinformationen benötigen. Dies ist noch kein vollwertiges Feature und wird wahrscheinlich noch wachsen.
  • Eine kurze "Wusstest du schon?"-Meldung nach dem Laden wurde hinzugefügt, um den Spielern zu helfen, unser System besser zu nutzen.

Bugfixes

  • Gegenstandssets sollten wieder richtig funktionieren (/Pakete) (erfordert das nächste Coremodul-Update!).
  • Die Option zum Erweitern der Modifikatorenliste für Würfe sollte nun wie vorgesehen funktionieren.
  • Ausrüstungsgegenstände sollten nun während der Migration in Containern bleiben.
  • Bei der Charaktererstellung hinzugefügte Muttersprachen haben jetzt keine Punktekosten. Dies führt zur korrekten Berechnung der freien Sprachpunkte, die in Professionen enthalten sind.
  • Ein kleines Problem mit der Darstellung von Dice so Nice im Konfigurationsmenü wurde behoben.
  • Kampfzauber sollten nun den korrekten -4 Modifikator auf Ausweichen anwenden.

Patch 2.2.1

Verbesserungen

  • Gruppensammelproben wurden etwas verbessert.
  • Es gibt nun ein weiteres Notizfeld, welches nur für den Besitzer des Charakterblatt sichtbar ist (damit kann man z.B. die Hintergrundgeschichte vor anderen Spielern verbergen).
  • Webm Token sollten im Chat nun richtig angezeigt werden.
  • Der Ausweichwert kann nun zum Verteidigungswert umgeschalten werden für Kreaturen. Dadurch bekommen diese den Malus durch mehrfache Verteidigungen nicht mehr. Wir aktualisieren alle betroffenen Kreaturen zu einem späteren Zeitpunkt.

Bugs

  • Das Wiederholen des Schadenswurfs bei Zaubern sollte wieder richtig funktionieren.
  • Professionen mit Eigenschaftsvorraussetzzungen auf NSC Charakterblätter zu ziehen, sollte nun die modifizierten Eigenschaftswerte richtig anzeigen.
  • Die Sonderfertigkeit "Angriff verbessern" sollte nun richtig funktionieren.
  • Tricks und Ausbildungsaufsätze für Tiere sollten nun dem Kreaturenblatt hinzufügbar sein (fehlende Active Effects brauchen dennoch ein Update des Tiergefährten Moduls zu einem späteren Zeitpunkt).
  • Die Funktion von Merkmalskenntnis und -focus sollten nicht mehr vertauscht sein.
